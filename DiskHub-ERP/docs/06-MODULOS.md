## Estrutura Interna Requerida por Módulo

Cada módulo no código ou na especificação de produto deve conter:

- **Objetivo**: Propósito de negócio do módulo.
- **Telas**: Lista de interfaces gráficas.
- **Fluxos**: Diagramas de caso de uso e transição de telas.
- **Banco de Dados**: Modelagem de tabelas e campos envolvidos.
- **APIs**: Endpoints REST consumidos ou disponibilizados.
- **KPIs**: Indicadores chave de performance (Margem, Conversão, CAC).
- **Relatórios**: Modelos de exportação.
- **Permissões**: Restrição de acesso por perfil.
- **Roadmap**: Evolução futura de features do módulo.
